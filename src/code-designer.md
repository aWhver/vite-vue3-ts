# Code Designer

从入口文件出发，分析依赖、创建设计、完成Vue3开发。

## 使用场景

- 用户提供一个文件路径，要求分析其依赖并开发
- 需要从现有代码提取设计思路
- 基于入口文件进行功能扩展或开发

## 工作流程

### 1. 读取入口文件和设计文档

接受用户提供的：
- 入口文件路径
- designer/[需求名称] 目录路径

读取该目录下的md文档：
- **需求说明** (`<需求名称>/requirements.md`): 必有，包含需求名称和详细需求描述
- **设计文档** (`<需求名称>/design.md`): 可选，架构设计、模块划分、接口设计等

## 文件读取优先顺序

当分析依赖时，按以下优先级读取：

1. **.vue** - Vue组件文件（最高优先级）
2. **.ts / .tsx** - TypeScript文件
3. **.js / .jsx** - JavaScript文件
4. **其他** - 配置文件、样式文件等

### 2. 分析依赖关系

分析入口文件的import/require语句，找出所有相关依赖：

- **跳过**: node_modules、npm包（包括@aurora等内网npm包，只跳过其源码分析）
- **包括**: 项目本地的相对路径文件

使用以下策略查找依赖：
1. 分析 import/require/import() 语句
2. 对于相对路径文件（./、../、@/、src/开头），在项目目录下查找
3. 递归分析直到没有新的本地依赖文件
4. **跳过所有npm包**（包括@aurora等内网包），不分析其源码

### 3. 代码开发

根据入口文件功能、需求说明和设计文档，完成代码开发工作。

## 输入结构

```
project-root/
├── designer/
│   └── <需求名称>/
│       ├── requirements.md  # 需求说明（必有）
│       └── design.md        # 设计文档（可选）
└── [源代码文件...]
```

## 注意事项

- @aurora 是内网npm包，需要从内网或项目本地查找
- 优先读取.js/.ts/.jsx/.tsx等源码文件
- 保持设计文档与实际代码的一致性
